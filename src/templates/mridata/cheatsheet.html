Keyword Usage Notes
Uploader uploader: <uploader> ,
uploader: me 		Keywords and values are case insensitive .
"me" returns all the datasets that you uploaded.
Tags tag: <tag 1> and <tag 2> ,
tags: <tag 1> and <tag 2> ,
tag: <tag 1> 		Keywords and value are case insensitive.
Inserting "and" will allow you to search for multiple tags. tags and tag are the same.
Project project: <project name> , Keywords and value are case insensitive.
Anatomy anatomy: <anatomy> , Keywords and value are case insensitive.
References references: <references> ,
ref: <references> 		Keywords and value are case insensitive.
Ref and References are the same thing, but ref is a shortcut for references.
Comments comments: <comment> Keywords and value are case insensitive.
Furthermore, it does NOT have to be an exact match, but rather if it contains.
Funding Support funding_support: <supporter> ,
funding: <supporter> ,
support: <supporter> , 		Keywords and value are case insensitive.
"funding" and "support" are both shortcuts for "funding_support".
System Vendor system_vendor: <vendor> ,
vendor: <vendor> 		Keywords and value are case insensitive.
"vendor" is a shortcut for "system_vendor".
System Model system_model: <model> ,
model: <model> 		Keywords and value are case insensitive.
"model" is a shortcut for "system_model".
Protocol Name protocol_name: <protocol> ,
protocol: <protocol> 		Keywords and value are case insensitive.
"protocol" is a shortcut for "protocol_name".
Coil Name coil_name: <coil> ,
coil: <coil> 		Keywords and value are case insensitive.
"coil" is a shortcut for "coil_name".
Sequence Type sequence_type: <sequence> ,
sequence: <sequence> 		Keywords and value are case insensitive.
"sequence" is a shortcut for "sequence_type".
UUID uuid: <uuid> ,
id: <uuid> 		Keywords and value are case insensitive.
"id" is a shortcut for "uuid".
Fully Sampled fullysampled: <yes or no> ,
sampled: <yes or no> 		Keywords and value are case insensitive.
If sampled is unknown, state "unknown" or leave it blank.
Response can also be "y" for "yes" and "n" for "no".
Responses can also take the form of "1" for "yes" and "0" for "no".
"sampled" is a shortcut for "fullysampled".
